# 10th-AYU-SpringBoot

### Spring Boot 워크북 수행 및 미션 제출을 위한 저장소입니다.

---

## 🛠️ 워크북 수행 과정
1. Fork: 본 레포지토리를 개인 계정으로 Fork 합니다.
2. Clone: Fork한 레포지토리를 로컬 환경에 Clone 합니다.
3. Open: IntelliJ IDEA를 사용하여 프로젝트를 열어줍니다.
4. Mission: 주차별 워크북 및 미션 내용을 확인하고 코드를 작성합니다.
5. Push: 완료된 코드를 본인의 Fork된 레포지토리에 Push 합니다.
6. PR: 원본 레포지토리로 **Pull Request(PR)**를 생성하여 제출합니다.

## 🔍 커밋 메시지 유형
> 일관된 프로젝트 관리를 위해 아래의 커밋 유형을 준수해 주세요.

| **유형**               | **설명**                         | 
|:---------------------|:-------------------------------| 
| **feature**          | **새로운 기능 추가**                  | 
| **refactor**         | **코드 리팩토링**                    | 
| **fix**              | **일반적인 버그 수정**                 | 
| **docs**             | **문서 수정 (README, 주석, 위키 등)**   | 
| **test**             | **테스트 코드 추가**                  |
| **chore**            | **기타 관리 작업 (라이브러리 업데이트 등)**    | 
| **comment**          | **주석 추가 및 변경**                 | 
| **rename**           | **파일/폴더명 수정 및 위치 이동**          | 
| **remove**           | **파일 삭제**                      |
